## Sophie Voice Matrix Debugging Session [15:11-15:22 EST]

### Problem Identified
- Aaron can call Sophie, Sophie responds with TTS
- BUT Sophie can't understand what Aaron says
- Whisper transcribes garbage: "I'm sorry", "I'm not gonna talk", "you" (hallucinations)

### Root Cause Found
- Audio from Element X **IS SFrame encrypted**
- We had **disabled SFrame decryption** on 2026-03-04 thinking LiveKit handles it
- LiveKit does NOT decrypt SFrame for E2EE calls - frames arrive encrypted
- Result: Feeding encrypted audio bytes to Whisper → hallucinations

### Debug Evidence
- Sent Aaron audio clips of what Sophie received
- First clip: Aaron confirmed "That is definitely encrypted"
- Audio analysis showed speech-like patterns (autocorr 0.39, voice freq ratio 6.1) but Whisper couldn't understand
- Debug files saved to `/tmp/sophie_debug_utterance_*.wav`

### Fixes Applied
1. Re-enabled SFrame decryption in `_process_single_frame()`
2. Added diagnostic logging to verify decryption is being attempted
3. Fixed syntax error (duplicate line causing IndentationError)
